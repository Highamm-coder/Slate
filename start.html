Make a slate.

A slate is a single HTML file — twelve slides in one document. You don't build one from scratch; you hand a template to an LLM, tell it your topic, and it adapts it. Three steps:

01

Get the template

Download the Slate template. It's a complete, working twelve-slide slate with placeholder copy in every slide type — the LLM's job is to swap the copy, not rebuild the design.

Download template Preview in browser
02

Give it to an LLM

Open Claude, Cowork, ChatGPT, or any capable LLM. Attach (or paste) the template file and include a short instruction. Here's one that works:

Adapt this Slate template to create a slate about [YOUR TOPIC]. Audience: [WHO THE DECK IS FOR]. Accent colour: [HEX, e.g. #3D3FB7]. Brand wordmark: [e.g. DEKSIA — replaces "STUDIO" in every slide's chrome]. YOU MAY CHANGE: — The three colour variables in :root (--accent, --accent-soft, --slide-bg-dark). — The <title> tag and the "STUDIO" wordmark in every chrome. — The number, order, and mix of slide types — add, remove, or reorder <section class="slide"> blocks to fit the story. — All copy inside each slide (headlines, body, labels, stats, quotes, attribution). KEEP EXACTLY AS-IS: — Every CSS rule below the :root block — typography, layout, chrome, meta row, and all slide-type styles. — Every class name (.slide, .dark, .eyebrow, .meta-row, .chrome, .inner, etc.). — The 16:9 aspect ratio and the cqw sizing system. — Inter as the only font. — The hairline-rule dividers (no cards, no thick borders, no shadows on text). — The bottom chrome on every slide and the meta row on every non-title slide. Return the complete modified HTML as a single code block.
03

Save, polish, present

Save the LLM's response as slate.html. Drop it on the Slate viewer to present — and click Edit in the topbar to make quick visual fixes (headlines, copy, slide order) without touching the code.

Prefer to paste instead of attach? Open the template in your browser, view-source (⌘⌥U / Ctrl-U), copy all, paste into chat alongside the instruction above.
